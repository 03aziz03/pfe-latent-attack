## A.1 The bug

results/metric_comparison.md shows mAP_drop@0.5 = 1.000 for ALL THREE attacks
(latent, pgd, fgsm). This is not all-three-attacks-being-perfect; it's a
trivial coordinate mismatch:

- data/images_50/*.jpg are at ORIGINAL DETRAC resolution (typically 960×540)
- results/adv_{latent,pgd,fgsm}/*.png are saved at 640×640 LETTERBOXED
  (with gray bars top/bottom — visible in f07/f08/f09)

Running YOLO on these without normalizing the frame size produces:
- dets_clean.json: boxes in (960, 540) coordinate system
- dets_*.json    : boxes in (640, 640) letterboxed coordinate system

Box IoU between these is ~0 by construction → mAP_drop trivially equals 1.0.

## A.2 The fix

Modify scripts/run_full_eval.py to make the two coordinate spaces match:

Option 1 (recommended): re-letterbox the clean images to 640×640 BEFORE
running detection on them, so both clean and adv detections are in the
same letterboxed frame. Implementation:

  from src.viz.letterbox import letterbox_image  # NEW helper
  for clean_path in sorted(Path("data/images_50").glob("*.jpg")):
      img = cv2.imread(str(clean_path))
      img_lb, scale, pad = letterbox_image(img, target=(640, 640))
      cv2.imwrite(str(tmp_dir / clean_path.name), img_lb)
      # also save scale, pad to a JSON sidecar for un-letterboxing later

Then run YOLO on tmp_dir/ instead of data/images_50/. All four detection
JSONs will then be in (640, 640) frame.

Add a unit test in tests/test_letterbox.py that round-trips a known box
through letterbox + un-letterbox and recovers within 1 pixel.
